El Problema

Actualmente el front-end de time tracking se basa en archivos de interfaz, cada uno de estos archivos puede contener:

  • Querys (MySQL)
  • PHP (print, scripts y clases)
  • Html
  • Javascript
  • ... y todas las convinaciones posibles de lo anterior.

Nueva Arquitectura

La nueva arquitectura propuesta por Daniel Ochoa, nos presenta un modelo separado en capas:

  • Business
  • Service
  • DAO
  • Persistent Media

Vista - Controlador

Son 3 nuevas capas que se agregan a la Arquitectura y son la que intectuan con el usuario.

  • Router
  • Controller
  • View

Nueva Arquitectura

Capa Router

El router es el encargado de asignar la tarea al controlador correspondiente según la URL.

Estas URL son formadas de la siguiente manera.


/ app / ControllerName / action / arg1 / arg2 / ...

  • ControllerName es el nombre del controlador sin sufijo Controller.
  • action es el méthodo del controlador que ejecutará la tarea.
  • arg... son lo argumentos que se pasarán a método

Capa controller

El controlador es quien transará con el negocio (Business) y entregará el resultado a la vista.

Por defecto el controlador llamará a la vista con el mismo nombre de la acción, pero esto puede ser cambiado con la acción render(), donde se le indica a que vista se le pasarán los datos.

Capa View

La vista es la encargada de entregarle la respuesta al usuario y se divide principalmente en 4 herramientas:

  • Layout
  • View
  • Element minivistas que puedes ser incluidas en cualquier vista
  • Helper clases que nos ayudan a construir el Html
